Recently, I am learning the usage of Apache Flink. In this note, I would like to write down some notes of Flink and distributed computing. (this page is updating)

##Basic

###What is Apache Flink

Apache Flink is an open source stream processing framework, which is a distributed streaming dataflow engine written in Java and Scala.

###Domain

Event-driven Applications【事件驱动】
Data Analytics Applications【数据分析】
Data Pipeline Applications【管道式ETL】

###Compare with Apache Spark(Spark Streaming)

Micro-Batching Distributed Snapshots
Implantation Apache Spark Apache Flink
Semantics Exactly-Once (To process every message once and only once) Exactly-Once
Delay High Low
Storm(吞吐) High High
Computing Model Micro Batch Streaming
Fault Tolerant Low Low
Flow control Relatively Bad Good
Business and Fault-tolerance Relatively Coupling Separation
Continuous data streams should not be split to the record level, but converged into a batch of micro-batch, atomic data for batch-like calculations. Determine the state of the current flow calculation (including the data being processed, and the Operator state), generate a consistent snapshot of the state, and store it in persistent storage
